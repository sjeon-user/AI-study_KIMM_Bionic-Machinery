1 터미널에서 실행 (CLI)

명령어설명
claudeClaude Code 시작
claude -c / --continue현재 폴더에서 가장 최근 대화 이어하기
claude -r / --resume세션 목록에서 선택하거나 특정 세션 이어하기
claude -r "세션이름"이름/ID로 특정 세션 바로 열기
claude -p "질문"한 번만 질문하고 종료 (SDK/스크립트용)
claude --debug디버그 로그 켜고 시작

2 세션 관리 (대화 안에서 / 입력)

명령어설명

/rewind / /undo이전 체크포인트로 되돌리기

8 팁

  • /만 입력하면 사용 가능한 전체 명령 목록이 나옵니다.
  • 명령어는 메시지 맨 앞에 써야 인식됩니다.
  • CLAUDE.md는 폴더에서 claude 실행 시 자동으로 읽힙니다. (아래 9절 참고)
  • -c / --continue = 최근 세션 바로 이어가기, -r / --resume = 목록에서 고르기.

9 CLAUDE.md 생성 가이드 (전역 vs 프로젝트)

CLAUDE.md는 Claude에게 매 세션마다 자동으로 전달되는 지침서입니다. 같은 말을 반복하지 않도록 규칙·스타일·워크플로를 적어 두는 파일입니다.

9-1. 전역 vs 프로젝트 비교

구분전역 (User)프로젝트 (Project)
위치~/.claude/CLAUDE.md./CLAUDE.md 또는 ./.claude/CLAUDE.md
Windows 예시C:\Users\내이름\.claude\CLAUDE.md...\내프로젝트\CLAUDE.md
적용 범위내 PC의 모든 프로젝트해당 폴더만
공유나만 (Git에 안 올림)팀과 Git으로 공유 가능
적합한 내용개인 코딩 스타일, 선호 도구, 공통 습관빌드/테스트 명령, 프로젝트 구조, 팀 규칙
생성 방법직접 작성 또는 /memory/init 또는 직접 작성
추가: 개인용 프로젝트 설정만 따로 두려면 CLAUDE.local.md (프로젝트 루트, .gitignore 권장)

9-2. 로드 순서 (넓은 범위 → 좁은 범위)

세션 시작 시 아래 순서로 합쳐져서 읽힙니다. 뒤에 오는 내용이 더 구체적입니다.

  • 조직 정책 (회사 배포용, IT 관리)
  • 전역 ~/.claude/CLAUDE.md
  • 상위 폴더의 CLAUDE.md (모노레포 등)
  • 프로젝트 ./CLAUDE.md 또는 ./.claude/CLAUDE.md
  • 같은 폴더의 CLAUDE.local.md (개인용)
현재 작업 폴더에서 위로 올라가며 CLAUDE.md를 찾습니다.
예: foo/bar/에서 실행 → foo/CLAUDE.md → foo/bar/CLAUDE.md 순으로 로드

9-3. 전역 CLAUDE.md 만들기

언제 쓰나: 모든 프로젝트에 공통으로 적용할 개인 규칙

# 1) 폴더 생성 (없으면)
mkdir %USERPROFILE%\.claude

# 2) 파일 생성 후 편집
notepad %USERPROFILE%\.claude\CLAUDE.md

또는 세션 안에서 /memory → 전역 CLAUDE.md 항목 선택해 편집

작성 예시 (전역)

# 개인 공통 규칙

- 응답은 한국어로 작성한다.
- 커밋 메시지는 한 줄 요약 + 이유 형식으로 쓴다.
- 테스트 없이 코드만 제안하지 말고, 실행 방법도 함께 알려준다.
- 불필요한 주석·과한 리팩터링은 하지 않는다.

9-4. 프로젝트 CLAUDE.md 만들기

언제 쓰나: 이 저장소/폴더에서만 지켜야 할 팀 규칙, 실습 규칙

방법설명
/init코드베이스를 분석해 초안 자동 생성 (이미 있으면 개선 제안만)
직접 작성프로젝트 루트에 CLAUDE.md 파일 생성
/memory로드된 메모리 파일 목록에서 프로젝트 CLAUDE.md 편집
 
# 프로젝트 폴더로 이동
cd Desktop\kimm-rnd-demo

# Claude Code 시작
claude

# 세션 안에서
/init

저장 위치 (둘 중 하나)

내프로젝트/ ├── CLAUDE.md ← 루트에 두기 (가장 흔함) └── .claude/ └── CLAUDE.md ← .claude 폴더에 두기

작성 예시 (프로젝트 — 실습용)

# 작업 규칙 (한국기계연구원 R&D 보고서)

- 측정값은 자료에 있는 값만 사용하고, 없는 값은 "확인 필요"로 표시한다.
- 보고서 문체는 "~함" 체, 단위는 SI 우선.
- 전문 약어는 처음 나올 때 풀어 쓴다(예: FFT = 고속 푸리에 변환).
- data/ 폴더의 원본 시험 데이터는 수정하지 않는다.
- 보고서는 results/ 폴더에 저장한다.

9-5. 잘 쓰는 법

넣으면 좋은 것

  • 빌드·테스트·실행 명령 (npm test, pytest 등)
  • 폴더 구조와 파일 역할
  • 코딩·문서 스타일 (문체, 단위, 네이밍)
  • "절대 하지 말 것" (원본 데이터 수정 금지 등)
  • 자주 쓰는 워크플로

넣지 말 것

  • 한 번만 필요한 긴 절차 → 스킬(skills)로 분리
  • 특정 파일에만 해당하는 규칙 → .claude/rules/로 분리
  • 너무 긴 설명 (파일당 200줄 이하 권장)

작성 팁

  • 모호하게: "코드 잘 정리해줘" ❌
  • 구체적으로: "src/api/ 아래는 2칸 들여쓰기" ✅
  • 충돌하는 규칙이 없게 주기적으로 정리

9-6. 관련 명령어 · 파일

항목용도
/init프로젝트 CLAUDE.md 초안 자동 생성
/memory전역·프로젝트 CLAUDE.md 목록 보기 및 편집
CLAUDE.local.mdGit에 안 올리는 개인용 프로젝트 설정
.claude/rules/*.md주제별·경로별 규칙 분리 (대형 프로젝트)
@파일경로CLAUDE.md 안에서 다른 파일 내용 가져오기

대화로 추가하기

이 규칙을 CLAUDE.md에 추가해줘: 보고서는 results/ 폴더에 저장한다.

9-7. 자주 묻는 것

질문답

10 Skill vs Connector 개념

Claude Code를 확장하는 방법은 크게 세 가지입니다. 역할이 다르므로 구분해 두면 좋습니다.

CLAUDE.mdSkillConnector
한 줄 정의매 세션 지침서반복 워크플로·절차 묶음외부 도구·서비스 연결
형태마크다운 파일SKILL.md + 폴더MCP 서버 (Connector)
누가 만드나사용자/팀사용자/팀 (또는 번들)서비스 제공자·Anthropic Directory
로드 시점세션 시작마다필요할 때만세션 시작 시 도구 목록 등록
호출 방법자동 적용/스킬이름 또는 자동 선택Claude가 도구(tool)로 호출
예시"보고서는 ~함 체"hwpx 생성 절차, 배포 체크리스트GitHub, Linear, Sentry, DB

10-1. Skill (스킬)이란?

Skill은 Claude에게 특정 작업 방법을 가르치는 재사용 가능한 지침 패키지입니다.

  • SKILL.md 파일에 YAML frontmatter + 마크다운 지침을 작성
  • /스킬이름으로 직접 실행하거나, 상황에 맞으면 자동으로 불러옴
  • CLAUDE.md와 달리 쓸 때만 컨텍스트에 로드 → 긴 절차·체크리스트에 적합

저장 위치

범위경로적용
개인 (전역)~/.claude/skills/스킬이름/SKILL.md모든 프로젝트
프로젝트.claude/skills/스킬이름/SKILL.md해당 프로젝트만
(구형).claude/commands/이름.mdskills와 동일하게 /이름으로 동작

폴더 구조 예시

.claude/skills/hwpx/ ├── SKILL.md ← 필수 (지침 + frontmatter) ├── template.md ← 선택 (템플릿) └── scripts/ ← 선택 (실행 스크립트)

SKILL.md 최소 예시

---
description: hwpx 보고서를 생성할 때 사용. 사용자가 hwpx, 한글 문서 생성을 요청할 때.
---

1. 보고서_양식.md를 참조한다.
2. results/ 폴더에 저장한다.
3. 문체는 "~함" 체를 따른다.

실습 예 (5교시 hwpx 스킬)

10-2. Connector (커넥터)란?

Connector는 Claude Code를 외부 도구·서비스·데이터에 연결하는 통합입니다.

  • 기술적으로는 MCP(Model Context Protocol) 서버
  • UI·설정에서는 Connector라고 부름 (claude.ai → Settings → Connectors)
  • 연결되면 Claude가 도구(tool)로 GitHub 이슈 조회, DB 쿼리, Sentry 에러 확인 등을 직접 수행
한 줄로: Skill = "어떻게 일할지 가르치는 레시피", Connector = "밖의 세계와 손·발"

연결 방법 (3가지)

방법설명

10-3. Skill vs Connector — 선택 가이드

같은 말을 매번 반복한다 → CLAUDE.md에 규칙 추가 긴 절차·체크리스트가 있다 → Skill (.claude/skills/) 외부 시스템 데이터가 필요하다 → Connector (MCP) 둘 다 필요하다 → Skill + Connector 조합
조합 예: hwpx Skill로 "보고서 작성 절차"를 가르치고, GitHub Connector로 PR·이슈를 직접 조회

10-4. 자주 묻는 것

질문답
Skill과 slash command 차이?.claude/commands/는 구형. skills로 통합됨. 동작은 같고 skills가 기능 더 많음
Connector 추가했는데 안 보여요claude.ai 계정으로 /login 확인. API 키 모드면 connector 미로드
MCP와 Connector 차이?같은 것. MCP = 프로토콜 이름, Connector = 사용자-facing 이름
Skill이 CLAUDE.md를 대체하나?아님. CLAUDE.md = 항상 적용되는 사실·규칙, Skill = 필요 시 로드되는 절차
Connector마다 매번 승인?/permissions로 allow 규칙 설정 가능. .mcp.json은 최초 1회 승인
